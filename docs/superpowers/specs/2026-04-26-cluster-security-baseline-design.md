# Cluster Security Baseline — Design Spec

**Date:** 2026-04-26
**Status:** Approved (auto mode)
**Scope:** Consul + Nomad gossip encryption and ACL bootstrap, idempotent.

## Problem

The current bootstrap brings up Consul and Nomad with no gossip encryption and no
ACLs. Any host on the cluster network (or anyone who reaches the agent ports)
can join the gossip pool, read state, and submit jobs. This is acceptable for a
demo but blocks the cluster from being left running.

## Goal

After `bin/bootstrap` completes:

1. Consul agents communicate with a shared gossip encryption key.
2. Consul ACLs are bootstrapped; default policy is `deny`; agent and Nomad
   tokens are issued with least-privilege policies.
3. Nomad agents use a gossip encryption key.
4. Nomad ACLs are bootstrapped; the management token is captured locally; an
   anonymous policy of `deny` is set; an `ingress` policy/token is created for
   the Traefik job.
5. Re-running `bin/bootstrap` does not regenerate keys, does not re-bootstrap
   ACLs, and does not break running services.

## Non-goals

- mTLS between agents (separate effort; gossip + ACL is the high-leverage step).
- Vault integration. Tokens are written to gitignored files on the operator
  workstation.
- Token rotation. Out of scope; documented as a follow-up.

## Architecture

### Secret material

Generated once on the operator workstation by a new `ansible/playbooks/secrets.yml`
play that runs **before** `bootstrap.yml`:

- `consul_gossip_key` — 32-byte base64
- `nomad_gossip_key`  — 32-byte base64
- `consul_bootstrap_token` — captured from `consul acl bootstrap` on first run
- `nomad_bootstrap_token`  — captured from `nomad acl bootstrap` on first run
- `nomad_ingress_token` — created via the bootstrap token, scoped to the
  `traefik` and `whoami` jobs

All of the above are persisted to `ansible/inventory/group_vars/secrets.yml`
(gitignored). On re-runs the playbook reads existing values instead of
regenerating.

### Role changes

- `roles/consul`:
  - Template `consul.hcl.j2` reads `consul_gossip_key`, sets
    `acl { enabled = true, default_policy = "deny", enable_token_persistence = true }`,
    and configures the agent token after bootstrap.
  - Handler restarts agent on key/ACL config change.
  - Tasks: bootstrap ACL on `servers[0]` only, register output in a fact, write
    back to `secrets.yml`.

- `roles/nomad`:
  - Template `nomad-server.hcl.j2` and `nomad-client.hcl.j2` read
    `nomad_gossip_key`, set `acl { enabled = true }`, and configure the
    Consul token for service registration.
  - Tasks mirror Consul: bootstrap ACL on `servers[0]`, persist token.

- `roles/traefik`:
  - Job template gets `consul_token` env so the Consul Catalog provider can
    read services under ACL.

- `roles/sample_app`:
  - Job submission uses `nomad_ingress_token`.

### Idempotency

Each ACL bootstrap task is wrapped in a `creates`/`when` guard keyed on the
presence of a token in `secrets.yml`. Gossip keys are generated by an Ansible
task that uses `lookup('ansible.builtin.password', ...)` with a fixed file path
under `ansible/inventory/.secrets/` so the same value is reused on re-runs.

### `bin/bootstrap` change

```
bin/bootstrap:
  render-inventory
  ansible-playbook secrets.yml      # NEW — generates/loads secret material
  ansible-playbook bootstrap.yml
```

## Failure modes

- ACL bootstrap returns 403 (already bootstrapped, token lost) — playbook fails
  loudly with remediation instructions to either restore `secrets.yml` from
  backup or `consul acl bootstrap reset` on the leader.
- Gossip key mismatch — agents fail to join; surfaced by the existing
  `wait_for_connection` task plus a new health check.

## Testing

- Smoke test extension (`tests/smoke/test_acl.sh`):
  - Unauthenticated `consul members` against the public IP returns ACL error.
  - Unauthenticated `nomad job status` returns ACL error.
  - Authenticated calls with the captured tokens succeed.
- Existing ingress smoke test continues to pass after the change.
