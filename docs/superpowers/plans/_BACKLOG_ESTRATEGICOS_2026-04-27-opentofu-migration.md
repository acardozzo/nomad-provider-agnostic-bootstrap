# OpenTofu Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate from Terraform 1.9.8 to OpenTofu (latest stable) with zero behavior change. Update CI to install OpenTofu, switch all `terraform` invocations in scripts and CI to `tofu`, and verify mocked-provider tests still pass.

**Architecture:** OpenTofu is a drop-in fork of Terraform 1.5.x with continued OSS development. The state file format is compatible. Provider ecosystem (vultr, linode, b2, cloudflare) works unchanged. Migration is limited to (a) installing OpenTofu in CI runners, (b) renaming `terraform` → `tofu` in scripts, (c) verifying state and tests.

**Tech Stack:** OpenTofu 1.10.x (the latest available as of plan date), GitHub Actions `opentofu/setup-opentofu`, existing Vultr/Linode providers, existing `cluster.tftest.hcl` mocked test.

## Task 4: Switch CI to OpenTofu

**Files:**
- Modify: `.github/workflows/lint.yml`

- [ ] **Step 1: Patch the terraform job**

Find the existing block:

```yaml
  terraform:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: 1.9.8
      - name: terraform fmt
        run: terraform -chdir=terraform fmt -check -recursive
      - name: terraform validate
        run: |
          terraform -chdir=terraform init -backend=false
          terraform -chdir=terraform validate

      - name: terraform test (mocked providers)
        run: terraform -chdir=terraform test
```

Replace with:

```yaml
  iac:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: opentofu/setup-opentofu@v1
        with:
          tofu_version: 1.10.6   # bump as needed; keep pinned

      - name: tofu fmt
        run: tofu -chdir=terraform fmt -check -recursive

      - name: tofu validate
        run: |
          tofu -chdir=terraform init -backend=false
          tofu -chdir=terraform validate

      - name: tofu test (mocked providers)
        run: tofu -chdir=terraform test
```

Replace the `terraform_version: 1.9.8` reference; rename the job from `terraform` to `iac` (semantically more accurate).

- [ ] **Step 2: Commit + push, watch CI**

## Task 7: New IaC runbook

**Files:**

- [ ] **Step 1: Write runbook**

```bash
cat > docs/runbooks/iac.md <<'EOF'
# Runbook — Infrastructure as Code (OpenTofu)

## Why OpenTofu, not Terraform
- BSL drama: HashiCorp 2023 license change made Terraform non-OSS.
- OpenTofu is the Linux Foundation fork; provider ecosystem (vultr, linode,
  b2, cloudflare) works unchanged.
- Adds features missing in Terraform: state encryption (1.7), provider
  iteration (1.8), early-eval variables (1.9).

## Pinned version
See `.github/workflows/lint.yml`. Bump deliberately: any new release should
be tested by re-running `tofu test` locally before pinning.

## Local install
```bash
brew install opentofu      # macOS
sudo snap install opentofu --classic   # Linux
```

## Daily commands
```bash
tofu -chdir=terraform init
tofu -chdir=terraform plan
tofu -chdir=terraform apply
tofu -chdir=terraform destroy
tofu -chdir=terraform test    # mocked-provider tests
```

## Rollback to Terraform (if needed)
1. Revert the commit `ci(iac): migrate from terraform 1.9 to opentofu 1.10`.
2. Revert `feat(bin): switch terraform calls to tofu`.
3. State files are forward-compatible: `terraform 1.9.8` will read state
   that OpenTofu wrote, **as long as no OpenTofu-specific features were
   used** (encryption, certain provider iteration patterns). Verify before
   rolling back.

## Provider version pins
`terraform/versions.tf` declares min versions. Bump deliberately: read the
provider CHANGELOG, run `tofu -chdir=terraform init -upgrade`, run
`tofu test`, commit the new lockfile.

## State storage
- Local default: `terraform.tfstate` in `terraform/`.
- Remote: see `terraform/backend.tf.example` (S3-compatible).
- Encryption-at-rest can be enabled via `terraform { encryption {} }` block;
  see https://opentofu.org/docs/language/state/encryption/ when activating.
EOF
```

- [ ] **Step 2: Commit**

```bash

git commit -m "docs(runbook): IaC tool choice and operations"
